Alternative orchestration to kubernetes

  • Jonathan Gonzalez
  • Published on 07/10/2019
  • DevOps

Due to different business reason I've been thinking on finding a viable alternative to kubernetes for those project that simply don't qualify for their usage. This is the result of my research, identifiying the different components:

 

Swarm GUI 
    - Option 1: OpenSource —> Portainer / Swarmpit
    - Option 2: Cloud Native / Enterprise —> Docker EE UCP
 
Central Monitoring 
    - Option 1: OpenSource —> Prometheus + Grafana 
    - Option 2: Cloud Native / Enterprise —> AWS Cloudwatch + Telegraph (or enterprise equivalent, like Sysdig Enterprise, DataDog, Librato)
 
Central Logging 
    - Option 1: OpenSource —> ELK 
    - Option 2: Cloud Native / Enterprise —> AWS CloudWatch + AWS CloudTrail (or enterprise equivalent, like Logz.io, Papertrail, etc…)
 
Container Runtime 
    - Single option —> Docker EE (prod) and Docker CE (dev/staging)
 
Container Registry
    - Option 1: OpenSource —> Docker Registry
    - Option 2: Cloud Native / Enterprise —> Docker Registry DTR
 
Layer 7 Proxy (aka Ingress Controller)
    - Single option —> HAProxy (or Traefik, or NGINX + Docker Interlock architecture*)
 
Orchestration 
    - Single option —> Docker Swarm
 
Storage (Block Storage)
    - Option 1: OpenSource —> REX-Ray + Volumes in cloud provider
    - Option 2: Cloud Native / Enterprise —> Docker CloudStor or REX-Ray or BlockBridge or Portworx or StorageOS or Diamanti (hyperconverged)
 
Networking 
    - Single option —> Docker Swarm Overlay
 
HW / OS 
    - Option 1: OpenSource —> Terraform / Ansible + Docker-Machine 
    - Option 2: CloudNative / Enterprise —> Terraform / Ansible + Docker-Machine + IaaS tools in cloud providers (i.e. AWS CloudFormation)
 
Security
    - Single option:
        - Host setup scanner —> Docker Bench
        - Container image scanner —> Aqua Microscanner
        - Container runtime scanner —> Sysdig Falco
        - System capturing & analysis —> Sysdig Inspect
 
(*) https://docs.docker.com/ee/ucp/interlock/architecture/
